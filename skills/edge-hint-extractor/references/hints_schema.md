# Hints Schema

Use this schema as input for downstream concept synthesis and auto detection.

```yaml
metadata:
  timestamp: "2026-02-22T12:00:00Z" # ISO 8601 format, UTC
  tool_version: "1.0.0" # Version of the edge-hint-extractor tool
  as_of: "2026-02-20"
  rule_hints: 6
  llm_hints: 3
  total_hints: 9
  regime: RiskOn
hints:
  - title: "Breadth-supported breakout regime"
    observation: "Risk-on regime with pct_above_ma50=0.65"
    hypothesis_type: "breakout"              # optional
    preferred_entry_family: "pivot_breakout"  # optional
    symbols: ["NVDA", "AVGO"]              # optional
    regime_bias: "RiskOn"                    # optional
    mechanism_tag: "behavior"                # optional
    context: "Market summary showed strong breadth" # optional, string describing the context
    line_number: 123 # optional, integer line number where the hint was found
```

## Field Descriptions

This section provides detailed descriptions for each field in the output YAML.

### Top-Level Fields

-   **`metadata`** (object): Contains information about the hint generation process.
    -   **`timestamp`** (string, required): The UTC timestamp when the hints were extracted, in ISO 8601 format (e.g., `"2026-02-22T12:00:00Z"`). This is crucial for traceability.
    -   **`tool_version`** (string, required): The version of the `edge-hint-extractor` tool used to generate these hints (e.g., `"1.0.0"`).
    -   **`as_of`** (string, required): The date for which the market observations and hints are relevant, in `YYYY-MM-DD` format (e.g., `"2026-02-20"`).
    -   **`rule_hints`** (integer, required): The count of hints generated purely by rule-based extraction.
    -   **`llm_hints`** (integer, required): The count of hints augmented or generated by an LLM.
    -   **`total_hints`** (integer, required): The total number of hints, which is the sum of `rule_hints` and `llm_hints`.
    -   **`regime`** (string, optional): The identified market regime (e.g., `"RiskOn"`, `"Neutral"`, `"RiskOff"`).

-   **`hints`** (array of objects, required): A list of extracted hint objects. Each object represents a single edge hint.

### Hint Object Fields

Each object within the `hints` array has the following fields:

-   **`title`** (string, required): A concise, human-readable summary or name for the hint (e.g., `"Breadth-supported breakout regime"`).
-   **`observation`** (string, required): A detailed description of the observation or signal that led to the hint (e.g., `"Risk-on regime with pct_above_ma50=0.65"`).
-   **`hypothesis_type`** (string, optional): Categorizes the type of hypothesis or market behavior the hint relates to.
    -   **Expected Values:** `breakout`, `earnings_drift`, `news_reaction`, `futures_trigger`, `calendar_anomaly`, `panic_reversal`, `regime_shift`, `sector_x_stock`.
    -   **Behavior:** Unrecognized values are first checked against keyword inference from the hint's title and observation; if no match is found, they fall back to `research_hypothesis`. Used for clustering when `--promote-hints` is enabled.
-   **`preferred_entry_family`** (string, optional): Suggests a preferred entry strategy or pattern.
    -   **Expected Values:** `pivot_breakout`, `gap_up_continuation`.
-   **`symbols`** (array of strings, optional): A list of specific stock symbols or assets the hint applies to (e.g., `["NVDA", "AVGO"]`). An empty list or absence means it's a broad market hint.
-   **`regime_bias`** (string, optional): Indicates the market regime bias associated with the hint.
    -   **Expected Values:** `RiskOn`, `Neutral`, `RiskOff`.
-   **`mechanism_tag`** (string, optional): A tag categorizing the underlying mechanism or driver of the hint.
    -   **Expected Values:** `behavior`, `flow`, `structure`, etc.
-   **`context`** (string, optional): Additional textual context or surrounding information from where the hint was extracted (e.g., `"Market summary showed strong breadth"`).
-   **`line_number`** (integer, optional): The line number in the original input file where the hint was identified. Useful for debugging and tracing.

## LLM CLI Contract

`build_hints.py --llm-ideas-cmd "<command>"` sends JSON to stdin:

```json
{
  "as_of": "YYYY-MM-DD",
  "market_summary": {...},
  "anomalies": [...],
  "news_reactions": [...],
  "instruction": "Generate concise edge hints ..."
}
```

The command must print either:

-   `[{...}, {...}]`
-   `{"hints": [{...}, {...}]}`
